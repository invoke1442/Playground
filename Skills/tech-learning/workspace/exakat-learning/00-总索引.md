# Exakat 学习索引

本目录围绕 Exakat 的 GitHub 仓库与官方文档，按 6 个主题拆成独立长文。每篇文档都尽量满足三个目标：

1. 面向非科班读者也能顺读。
2. 关键结论都能追到官方来源或源码位置。
3. 不只说“是什么”，还说“为什么这样设计”和“工程上怎么用”。

## 文档清单

1. `01-开源性与产品形态详解.md`
   重点讲清 Exakat 到底是不是开源、开源到什么程度、Community Edition / Enterprise / Cloud 的边界、AGPL 对使用者意味着什么。

2. `02-CLI-用法与参数详解.md`
   重点讲命令行入口、实际命令族、参数分层、常见组合、当前源码和官方文档的差异、如何避免把内部命令当成普通用户命令误用。

3. `03-规则仓库与规则集详解.md`
   重点讲规则源码放在哪里、规则集是如何组织出来的、为什么文档里的规则数量与当前 CE 仓库不一致、如何从源码与文档双向定位某条规则。

4. `04-静态分析原理与工作流详解.md`
   重点讲 Exakat 如何从 PHP 源码走到 AST、依赖图、Gremlin 图数据库、SQLite dump，再到报告生成；也解释 `project` 命令到底做了哪些阶段。

5. `05-常用规则语法与开发方式详解.md`
   重点讲 Analyzer 类、DSL 风格查询链、`dependsOn()`、`prepareQuery()`、常见 DSL 动词和一条规则从“想法”到“可运行”的落地方式。

6. `06-典型漏洞扫描工程最佳实践.md`
   重点讲把 Exakat 真正用于漏洞扫描和持续审计时，怎么选规则、怎么做范围控制、怎么减少噪音、怎么在 CI/CD 或大项目中落地。

## 统一假设

- 输出语言：中文。
- 读者水平：初学到中级，默认不预设静态分析、图数据库、编译器课程背景。
- 版本裁决：
  - 当前源码入口以 GitHub 仓库 `exakat/exakat-ce` 为准。
  - 本次本地实测版本为 `2.6.7`。
  - 官方在线文档中部分页面明显保留了较旧版本措辞，我在正文中会专门指出“当前源码”和“文档对外表述”之间的差异。

## 重要提醒

- Exakat 官方文档展示的是“Exakat 引擎/产品线”的全貌，里面经常同时包含 Community Edition、Enterprise Edition、Cloud 的信息。
- 当前 GitHub 仓库 `exakat/exakat-ce` 只代表 Community Edition 的代码基线，因此你会看到“官网说 1600+ 规则，而 CE 本地 catalog 只有 361 条规则”的现象。这不是简单的文档错误，而是“产品全量能力”和“开源子集能力”的混合展示。
- 所以本目录里所有涉及“数量”的结论，我都会明确标注是：
  - “官网/官方文档宣称的产品总量”
  - “当前 CE 仓库和 CLI 实测的开源子集数量”
